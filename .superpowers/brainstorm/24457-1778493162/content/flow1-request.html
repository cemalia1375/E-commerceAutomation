流程图 1：请求处理链路

从用户发消息到收到回复的完整路径

👤 用户 → POST /agent/chat
↓
① FastAPI Router (chat.py)
解析请求参数 → 提取 session_key, tenant_key, message
⬇️
② SessionStore.get_or_create()
- 冷启动：调用 MainAgent.make_context_builder() + make_tool_registry()
- 热启动：从 MySQL 恢复消息历史
- 注入 dynamic_context + attention_packets
⬇️
③ ReactLoop.run()
a. ContextBuilder.build() → 组装 system prompt（稳定+动态）
b. LLM.stream_with_retry() → 流式获取 token + tool_calls
c. 文本 token → 实时 TextEvent → SSE 推给前端
d. 有工具调用 → 区分耦合/解耦 → 执行
e. 耦合工具结果注入历史 → 回到 b
f. 无工具调用 → DoneEvent
⬇️
📤 SSE 流式响应 → 用户
⚡ 响应完成后：PostprocessHook / MemoryExtract / ColdPathHook → 异步后台任务
