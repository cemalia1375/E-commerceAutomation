流程图 3：Subagent 调度

主 Agent → 子 Agent → 后处理的完整链路

① ReactLoop 调用 Subagent 工具
主 Agent 的 LLM 选择调用 SkinDiary 或 DeepReport 工具 → 工具执行将任务提交到 SUBAGENT_DISPATCH Stream
⬇️
② SubagentStore.dispatch()
根据任务类型路由到对应 Subagent 实例：
- SkinDiarySubagent：创建独立的 ReactLoop + ContextBuilder + ToolRegistry
- DeepReportSubagent：同上，各自拥有独立的记忆和上下文
⬇️
③ Subagent ReactLoop 执行
子 Agent 独立执行 ReAct 循环，共享主 Agent 的合规约束（compliance.md）
执行完成后：
- 结果写入 MySQL（document_repo / skin_diary_result_repo）
- 提交后处理任务到 POSTPROCESS Stream
- 通过 EventHub 发布事件通知
⬇️
④ 后处理链路
PostprocessExecutor 消费任务：
1. PostprocessHook — 文档后处理（格式化/裁剪）
2. ColdPathHook — 结构化记忆提取（话题跟踪）
3. SkinProfileSync — 皮肤档案同步（增量更新）
4. MemoryExtract — 长期记忆提取 → MySQL memory 表
