# 客户端与跨集高光日志入库方案（确认版）

日期：2026-07-14

## 1. 目标

为 FlowCut 客户端 exe 和跨集高光任务链路补齐可查询、可审计、可排障的日志入库能力。范围包括：

- 客户端 exe 用户行为：启动、页面/tab、按钮点击、上传、接口失败、前端异常。
- 跨集高光链路：批次创建、取消、阶段流转、worker 执行、成片合成/导出。
- 外部 API 报错：moyu/Gemini/ASR/OSS/Qdrant 等调用失败、超时、响应格式异常。
- Agent 工具调用：继续复用 `nb_agent_tool_invocations`，后续补齐 FlowCut 接入。

本轮不覆盖千川/巨量广告投放发布，也不覆盖普通素材脚本自动化。

## 2. 落地原则

1. 新增两张表：`fc_client_event_log`、`fc_highlight_event_log`。
2. 表名已确认，不使用 `fc_highlight_audit_event` / `fc_highlight_pipeline_event`。
3. 现阶段只新增表和索引，不破坏已发 exe 依赖的旧字段。
4. 仅当字段完全未被后端、前端、release 打包文件引用时，才列入删除候选。
5. 全部关联使用普通字段和普通索引，不建外键，不写 `FOREIGN KEY` / `REFERENCES`。
6. 日志中禁止写入密码、cookie、session token、API key、OSS key、完整 prompt、完整模型响应、完整视频内容和用户本地绝对路径。

## 3. 字段删除评估

已检查当前代码引用，以下字段仍被后端、前端或高光运行链路使用，本轮不删除：

| 表 | 字段 | 结论 |
|---|---|---|
| `fc_highlight_asset` | `oss_url` | 仍被后端 fallback、前端展示和素材/成片逻辑引用 |
| `fc_highlight_batch` | `orchestrator_state_json` | 高光编排恢复、进度计算仍使用 |
| `fc_highlight_batch` | `summary_json` | 进度、错误摘要、最终结果仍使用 |
| `fc_highlight_batch` | `merged_shots_json` | 起点选择阶段仍读取 |
| `fc_highlight_stage` | `input_json` | span plan 等阶段读取候选输入 |
| `fc_highlight_stage` | `result_json` | 阶段结果、进度展示仍使用 |
| `fc_highlight_stage` | `error` | 失败原因展示与批次错误汇总仍使用 |
| `fc_creative` | `highlight_reason_json` | 前端详情、失败记录仍使用 |
| `fc_creative` | `compose_plan_json` | 成片合成/展示仍使用 |
| `fc_creative` | `clip_plan_json` | 合成执行、前端解析仍使用 |

结论：当前没有可安全直接删除的高光冗余字段。若未来要瘦身，应在新版本代码完成兼容迁移后再单独执行。

## 4. 新增表一：`fc_client_event_log`

用途：记录客户端 exe 用户行为和前端/API 请求结果。

```sql
CREATE TABLE IF NOT EXISTS fc_client_event_log (
    id              BIGINT       NOT NULL AUTO_INCREMENT,
    event_id        VARCHAR(64)  NOT NULL,

    tenant_key      VARCHAR(255) NOT NULL DEFAULT 'flowcut',
    user_id         BIGINT       NULL,
    username        VARCHAR(128) NULL,
    session_key     VARCHAR(255) NULL,

    app_version     VARCHAR(64)  NULL,
    platform        VARCHAR(64)  NULL,
    device_id       VARCHAR(128) NULL,

    event_type      VARCHAR(128) NOT NULL,
    event_source    VARCHAR(32)  NOT NULL DEFAULT 'client',
    status          VARCHAR(32)  NOT NULL DEFAULT 'ok',
    severity        VARCHAR(16)  NOT NULL DEFAULT 'info',

    page            VARCHAR(128) NULL,
    route           VARCHAR(255) NULL,
    component       VARCHAR(128) NULL,
    action          VARCHAR(128) NULL,

    request_id      VARCHAR(64)  NULL,
    http_method     VARCHAR(16)  NULL,
    api_path        VARCHAR(255) NULL,
    http_status     INT          NULL,
    duration_ms     INT          NULL,

    batch_id        VARCHAR(64)  NULL,
    entity_type     VARCHAR(64)  NULL,
    entity_id       VARCHAR(128) NULL,

    error_code      VARCHAR(128) NULL,
    error_message   VARCHAR(512) NULL,
    payload_json    JSON         NULL,

    ip_address      VARCHAR(64)  NULL,
    user_agent      VARCHAR(512) NULL,

    occurred_at     DATETIME     NOT NULL,
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,

    PRIMARY KEY (id),
    UNIQUE KEY uq_fc_client_event_id (event_id),
    KEY idx_fc_client_event_tenant_time (tenant_key, occurred_at),
    KEY idx_fc_client_event_user_time (user_id, occurred_at),
    KEY idx_fc_client_event_session_time (session_key, occurred_at),
    KEY idx_fc_client_event_type_time (tenant_key, event_type, occurred_at),
    KEY idx_fc_client_event_status_time (tenant_key, status, occurred_at),
    KEY idx_fc_client_event_batch_time (tenant_key, batch_id, occurred_at),
    KEY idx_fc_client_event_request (request_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
COMMENT='FlowCut 客户端 exe 用户行为日志';
```

建议事件类型：

```text
client.app.start
client.app.exit
client.tab.open
client.button.click
client.upload.select
client.upload.submit
client.upload.success
client.upload.failed
client.api.request.failed
client.api.request.timeout
client.page.error
highlight.batch.create.click
highlight.batch.cancel.click
highlight.creative.preview.open
highlight.creative.export.click
```

## 6. 现有表补充索引

本轮只补普通索引，便于排查和串联日志，不修改字段、不删除字段、不建外键。

```sql
ALTER TABLE fc_highlight_batch
  ADD KEY idx_fc_highlight_batch_tenant_time (tenant_key, created_at),
  ADD KEY idx_fc_highlight_batch_tenant_status_time (tenant_key, status, updated_at);

ALTER TABLE fc_highlight_stage
  ADD KEY idx_fc_highlight_stage_runtime_task (runtime_task_id),
  ADD KEY idx_fc_highlight_stage_creative (creative_id);

ALTER TABLE nb_runtime_tasks
  ADD KEY idx_nb_runtime_tasks_tenant_status_time (tenant_key, status, updated_at);
```

实际代码中应以幂等方式添加索引：若索引已存在则忽略错误。

## 7. 逻辑关联

以下全部是逻辑关联，不建立数据库外键：

```text
fc_client_event_log.batch_id -> fc_highlight_batch.batch_id
fc_highlight_event_log.batch_id -> fc_highlight_batch.batch_id
fc_highlight_event_log.runtime_task_id -> nb_runtime_tasks.task_id
fc_highlight_event_log.tool_invocation_id -> nb_agent_tool_invocations.invocation_id
fc_highlight_event_log.entity_type/entity_id -> asset/creative/stage 等业务实体
nb_agent_tool_invocations.business_ref_id -> fc_highlight_batch.batch_id
nb_agent_tool_invocations.runtime_task_id -> nb_runtime_tasks.task_id
```

## 8. Agent 工具调用追踪补齐

`nb_agent_tool_invocations` 已建表，但 FlowCut 的 `MainAgent` 还没有把 `invocation_store` 传进 `ToolRegistry/ToolExecutionRuntime`，所以当前不能依赖它完整记录“用户触发了哪个工具”。

后续需要补齐：

1. 新增或复用 `ToolInvocationRepository`。
2. `Flowcut.api.container.AppContainer` 增加 `tool_invocation_repo`。
3. `build_container()` 初始化 `tool_invocation_repo`。
4. 构造 `MainAgent` 时传入 `tool_invocation_store=tool_invocation_repo`。
5. `MainAgent.make_tool_registry()` 创建 `ToolRegistry` 时传入 `invocation_store`。

高光工具写入约定：

| 字段 | 写入建议 |
|---|---|
| `tool_name` | `create_cross_episode_highlights`、`check_task_status` |
| `tool_category` | 创建任务为 `async_task`，查询类为 `sync_read` |
| `runtime_task_id` | 逻辑关联 `nb_runtime_tasks.task_id` |
| `business_ref_type` | `highlight_batch` |
| `business_ref_id` | `fc_highlight_batch.batch_id` |
| `input_json` | `drama_name/num_candidates/connector_asset_id` 等入参摘要 |
| `output_summary` | 工具返回摘要 |

## 9. 查询示例

查询某个客户端用户最近操作：

```sql
SELECT occurred_at, event_type, status, page, component, action,
       api_path, http_status, duration_ms, error_message
FROM fc_client_event_log
WHERE tenant_key = 'flowcut'
  AND user_id = 目标用户ID
ORDER BY occurred_at DESC
LIMIT 100;
```

查询某个高光批次完整链路：

```sql
SELECT occurred_at, event_source, event_type, status, stage,
       runtime_task_id, tool_invocation_id, entity_type, entity_id,
       duration_ms, error_code, error_message, context_json
FROM fc_highlight_event_log
WHERE tenant_key = 'flowcut'
  AND batch_id = '目标_batch_id'
ORDER BY occurred_at ASC, id ASC;
```

查询 moyu/Gemini 中转失败：

```sql
SELECT occurred_at, batch_id, stage, runtime_task_id,
       external_provider, external_base_host, external_model,
       external_operation, external_http_status,
       error_code, error_message
FROM fc_highlight_event_log
WHERE tenant_key = 'flowcut'
  AND external_base_host = 'moyu.info'
  AND status IN ('failed', 'timeout')
ORDER BY occurred_at DESC
LIMIT 100;
```
