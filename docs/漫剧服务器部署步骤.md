> **当前部署范围：跨集高光功能**
>

## 一、前置准备

### 1.1 本地环境要求

- Docker Desktop（需支持 `linux/amd64` 架构镜像构建）
- SSH 客户端（连接测试服务器）
- Git（拉取最新代码）

### 1.2 .env 环境变量文件

部署前需确保 `SimpleClaw/.env` 文件已配置

> **注意：** `.env` 文件已在 `.gitignore` 中排除，不会提交到仓库。部署时直接打入镜像。

## 二、镜像构建

### 2.1 构建命令

在项目根目录（`E-commerceAutomation/`）执行：

```bash
docker build \
  --platform linux/amd64 \
  -t mj-v1.0.0 \
  -f Dockerfile.mj \
  .
```

**参数说明：**

| 参数 | 说明 |
|------|------|
| `--platform linux/amd64` | 指定目标架构，确保在测试服务器（x86_64）上可运行 |
| `-t mj-v1.0.0` | 镜像名称 + 版本号 |
| `-f Dockerfile.mj` | 使用漫剧专用 Dockerfile（含 .env 打入 + 前端构建） |
| `.` | 构建上下文为项目根目录 |

### 2.2 构建产物

构建完成后，镜像包含：

| 组件 | 说明 |
|------|------|
| Python 3.11 + FlowCut 后端 | uvicorn 监听 `127.0.0.1:8001` |
| Nginx | 反向代理 `/api/*` → 后端，静态文件服务，监听 `80` |
| FFmpeg | 视频处理（高光合片、音轨提取） |
| supervisord | 进程管理（nginx + uvicorn） |
| .env | 环境变量已打入镜像 |

### 2.3 验证构建

```bash
# 查看构建的镜像
docker images | grep mj

# 本地快速验证（需要先配置好 .env 中的数据库连接）
docker run --rm -p 8080:80 mj-v1.0.0
# 然后访问 http://localhost:8080 确认前端页面可加载
# 访问 http://localhost:8080/api/health 确认后端健康检查返回 200
```

## 五、常用运维命令

### 5.1 查看日志

```bash
# 实时日志
docker logs -f mj-v1.0.0

# 最近 100 行
docker logs --tail 100 mj-v1.0.0
```

### 5.2 重启服务

```bash
docker restart mj-v1.0.0
```

### 5.3 停止 / 启动

```bash
docker stop mj-v1.0.0
docker start mj-v1.0.0
```

### 5.4 更新镜像

```bash
# 1. 拉取新版本
docker pull agentv1n8nworkflow-cn-guangzhou.cr.volces.com/workflow/mj:v1.0.1

# 2. 停止并删除旧容器
docker stop mj-v1.0.0
docker rm mj-v1.0.0

# 3. 启动新版本
docker run -d \
  --name mj-v1.0.1 \
  --restart unless-stopped \
  -p 80:80 \
  agentv1n8nworkflow-cn-guangzhou.cr.volces.com/workflow/mj:v1.0.1
```

### 5.5 进入容器调试

```bash
docker exec -it mj-v1.0.0 bash

# 容器内检查进程
supervisorctl status

# 容器内手动测试后端
curl http://127.0.0.1:8001/health
```

### 5.6 清理旧镜像

```bash
# 查看镜像占用
docker images

# 删除旧版本镜像
docker rmi agentv1n8nworkflow-cn-guangzhou.cr.volces.com/workflow/mj:v1.0.0
```

## 七、故障排查

### 容器无法启动

```bash
# 查看详细日志
docker logs mj-v1.0.0

# 常见原因：
# 1. MySQL 连接失败 → 检查 MYSQL_HOST/PORT/USER/PASSWORD 是否正确
# 2. OSS 配置错误 → 检查 FLOWCUT_OSS_* 变量
# 3. 端口冲突 → 宿主机 80 端口已被占用
```

### 前端页面空白

```bash
# 检查 nginx 是否正常运行
docker exec mj-v1.0.0 supervisorctl status

# 检查静态文件是否存在
docker exec mj-v1.0.0 ls /usr/share/nginx/html/
```

### API 返回 502

```bash
# 检查 uvicorn 进程是否存活
docker exec mj-v1.0.0 supervisorctl status

# 查看 uvicorn 错误日志
docker logs mj-v1.0.0 2>&1 | grep -i error
```
