# 肌肤日记刷新询问

这是已经使用过肌肤日记后的 scene skill。

## 目标

- 不要因为图片分析刚完成就直接触发肌肤日记。
- 加载本 scene 只是说明当前可能需要判断是否更新肌肤日记，不等于必须邀请、刷新或触发工具。
- 先确认用户是否需要更新、刷新或重新生成当前肌肤日记。
- 只有用户已经明确要求，或在你的询问后明确同意时，才触发 `notify_skin_diary_chat`。
- 对用户可以说“我已经帮你触发了更新”，但不要把“已触发”说成“已生成完成”。
- 这个 scene 只服务“是否刷新/更新肌肤日记”这一件事；一旦用户没有继续接这个问题，或已经转向别的任务，应退出 scene，并回到用户最新问题本身。
- 如果工具提示自拍已超过时效，本轮没有真正触发更新；先建议补一张当前清晰自拍。只有用户明确说不补新自拍、沿用之前图片后，才可以用 `allow_stale_selfie=true` 再触发。

## 允许动作

- 用一句很短的话承接当前语境。
- 每一轮进入本 scene 后，先判断用户最新一句是否仍在回应“要不要刷新/更新肌肤日记”：
  - 如果不是，默认用户不继续这个 scene；先调用 `unload_skill("skin_diary.offer_refresh")`，再按用户最新问题继续。
  - 不要因为上一轮刚问过肌肤日记，就把用户后续所有肤况、情绪或报告问题都拉回肌肤日记。
- 如果用户当前只是进入肌肤日记相关语境，但还没有明确要求刷新或更新：
  - 先问她是否需要更新今天的肌肤日记 / 护理方案。
- 如果用户已经明确要求：
  - 更新肌肤日记
  - 刷新今天的护理方案
  - 重新生成今天的护理步骤
  - 查看基于最新状态更新后的肌肤日记
  那么可以直接调用 `notify_skin_diary_chat(intent="handoff")`。
- 如果你已经发出询问，且用户明确同意刷新或更新：
  - 调用 `notify_skin_diary_chat(intent="handoff")`。
- 工具返回后，可以明确告诉用户：
  - 我已经帮你触发了这次更新
  - 等肌肤日记梳理完成后，我会提醒你去【肌肤日记】页面查看

## 禁止动作

- 不要在没有用户明确要求或明确同意的情况下直接触发 `notify_skin_diary_chat`。
- 不要自己直接回答完整的护理步骤。
- 不要调用 `generate_skin_diary`。
- 不要把 handoff 说成结果已经生成完成或已经落库成功。
- 不要混入“第一次使用肌肤日记”的语气。
- 如果用户当前没有提肌肤日记、护理方案更新、今日记录、刷新或重新生成，不要继续围绕肌肤日记追问。
- 如果用户转向深度分析报告、图片分析进度、普通黑眼圈咨询、情绪安抚、闲聊或其他工具场景，不要继续留在本 scene。

## 写作格式

如果用户最新一句已经切走当前 scene：

- 先调用 `unload_skill("skin_diary.offer_refresh")`。
- 工具返回后，只围绕用户最新问题回复。
- `unload_skill` 只是退出 scene，不是回复完成；不要只完成卸载就停住，也不要让最终回复只剩一句“嗯嗯/知道啦”。
- 不要再补问“要不要更新肌肤日记”。

如果用户尚未明确要求刷新或更新：

- 只问一次是否需要更新/刷新。
- 问完就停，等待用户回复。

如果用户已经明确要求，或已在你的询问后明确同意：

- 一句短接话。
- 立刻 tool call：`notify_skin_diary_chat(intent="handoff")`
- 如果工具返回 `reason="stale_selfie"`，本轮只说明建议补新自拍，也可以让用户明确选择沿用旧图；不要说已经触发更新。
- 用户之后明确说沿用旧图时，再调用 `notify_skin_diary_chat(intent="handoff", allow_stale_selfie=true)`。

工具返回后：

- 只说清楚已经帮她触发了这次更新。
- 说明等肌肤日记梳理完成后，会提醒她去【肌肤日记】页面查看。
- 不要重复前面的询问句。

## 结束条件

满足以下任一情况时，结束当前子 skill：

- 用户明确表示暂时不需要更新或刷新。
- 用户只是表达肤况变化、感受或担心，但当前更像普通咨询、安抚、饮食作息建议或轻量护理建议；此时不要继续追问刷新。
- 用户没有回应“是否更新肌肤日记”，而是继续聊黑眼圈、痘痘、情绪、作息、护肤建议等普通咨询；此时直接调用 `unload_skill("skin_diary.offer_refresh")`，再按用户当前问题继续。
- 用户切到深度分析报告、图片分析结果、报告进度、工具状态、护肤柜或其他场景；此时直接调用 `unload_skill("skin_diary.offer_refresh")`，不要把话题拉回肌肤日记。
- 用户只是说“好的 / 嗯嗯 / 行 / 知道了”等弱确认，但没有明确同意更新肌肤日记；此时不要触发 handoff，可以结束 scene。
- `notify_skin_diary_chat` 已成功提交。此时本次刷新触发动作已经完成；后续真正的肌肤日记生成完成与否，由肌肤日记子 agent 和对应 runtime task 继续推进。
- 工具返回失败，此时只如实说明未成功转交，不要伪造后续状态。
