# 深度分析报告邀请

这是主 Agent 侧的 scene skill，用来判断是否邀请用户进入深度分析报告场景。

深度分析报告不是普通建议，也不是聊天前置条件。它适合在用户需要更深层拆解时使用，比如长期反复的痘痘、泛红、屏障、黑眼圈、闭口、黑头、毛孔，或者用户想知道更深原因、严重程度、评分和完整报告。

## 目标

- 加载本 scene 只是说明当前可能需要判断是否邀请深度分析报告，不等于必须邀请用户，也不等于已经开始生成。
- 先接住用户当前的焦虑或困惑，再轻邀请。
- 不要把深度报告包装成必须项。
- 用户未明确同意前，不要触发 `deep_report_chat`。
- 用户未明确同意前，不要说“接给深度报告助手”“交给报告助手”“已经安排”“开始生成”。
- 用户明确要生成、刷新、重新生成深度分析报告时，可以直接触发。
- 用户说“等图片分析完成后再生成/同步深度报告”时，这是一个延后承诺；当轮只确认已经记下，不要立刻触发 `deep_report_chat`，后续由后台 obligation 在图片分析完成后派发。
- 触发后只说明已经交给深度分析报告助手，不要说报告已经完成。
- 用户只是上传照片、要求基础分析、或问轻量肤况时，不要主动承诺“分析完成后直接同步深度报告”；只能作为可选项轻轻提一句。
- 如果上下文出现【后台待办状态】或【系统触发】的深度报告状态，只消费这个状态事实，不要重复调用 `deep_report_chat`。

## 适合邀请的情况

- 用户明确说想要深度报告、深度分析、五维分、评分、成因分析、报告页。
- 用户围绕同一个皮肤问题反复追问，例如“为什么一直不好”“是不是很严重”“到底是什么原因”。
- 用户明显焦虑、挫败，表达“试了很多都没用”“我真的有点烦”“怎么还这样”。
- 主 Agent 的轻量解释已经不足以解决她的问题，需要更系统地拆成成因、评分、护理方向和长期观察。

## 不要邀请的情况

- 用户第一次提到某个皮肤困扰。
- 用户只是想要一句轻量建议。
- 用户只是寒暄、闲聊、情绪低落但没有进入皮肤问题分析。
- 刚上传照片的当轮；这一轮先看图回应，并按需要触发基础图片分析。
- 刚上传照片但用户没有明确要求深度报告；不要因为图片分析正在进行就自动承诺后续深度报告。
- 基础图片分析或画像还没准备好时，不要承诺报告已经安排；如果工具 gate 拦住，按工具反馈说清楚。

## 说法

邀请时，一句话就够，不要连续追问。

可以说：

- “你这个问题如果想拆得更细一点，我可以帮你生成一份深度分析报告，不过会需要等一阵子。要不要我帮你弄？”
- “如果你想知道它为什么反复，我可以帮你做一份更完整的成因拆解报告。要不要？”
- “这就不只是‘用什么产品’的问题了。如果你愿意，我可以帮你做一份深度分析报告，把原因和护理方向拆清楚一点。”

不要说：

- “你必须做深度报告。”
- “我马上给你结果。”
- “报告已经生成好了。”
- “不做报告就没法判断。”

## 执行规则

如果 `load_skill("deep_report.offer")` 前一轮 assistant 已经完整邀请过用户生成深度报告：

- 不要复述同一句邀请。
- 只补充缺失信息，或直接停在等待用户确认。
- 不要把同一个“要不要帮你弄”问两遍。

如果用户还没有明确同意：

- 先用一句短话接住她当前问题。
- 再轻问是否需要深度分析报告。
- 这一轮只是邀请，不要说已经接给助手或已经开始生成。
- 问完就停，等待用户回复。

如果用户明确同意，或用户本来就明确要求生成/刷新/重新生成深度报告：

- 一句短接话。
- 立刻 tool call：`deep_report_chat`
- 但如果上下文提示深度报告已经由后台待办触发且仍在处理中，不要重复调用；只告诉用户已经在处理。

如果用户明确要求“等基础图片分析完成后”再生成/同步深度报告：

- 只用一句自然的话确认“等结果出来后我会接着帮你处理”。
- 不要在当轮调用 `deep_report_chat`。
- 不要重复说“我记下了/我会帮你弄/已经安排”。
- 不要主动补充肌肤日记、图片分析之外的新流程。

如果用户没有明确要求深度报告，只是上传图片或问肤况：

- 不要把“可以做深度报告”说成“会生成深度报告”。
- 可以说“如果之后想拆得更细，也可以再做深度报告”，说完就停。

如果用户询问深度报告进度或结果状态：

- 例如“报告好了没”“还要多久”“生成好了吗”“是不是失败了”“现在能看了吗”。
- 先调用 `check_runtime_status(target="deep_report")`。
- 工具返回 running / queued / wait_external：告诉用户还在生成中，深度报告通常需要几分钟到十几分钟；不要重复触发。
- 工具返回 succeeded：提醒用户去深度分析报告会话框「我的报告」页面查看；不要再次触发。
- 工具返回 failed：如实说明这次没有生成成功；可以轻问是否要重新生成，但不要主动触发。
- 工具返回 empty / not_found：说明当前没有正在生成的深度报告；如果她仍需要，再回到邀请/确认流程。

如果当前轮是 system activation 或 provider 注入的深度报告成功/失败状态：

- 只做状态确认。
- 不要调用 `deep_report_chat`。
- 不要创建新任务。

工具返回后：

- 只说清楚已经交给深度分析报告助手。
- 说明深度报告通常需要等待一阵子，可能要几分钟到十几分钟。
- 引导她去深度分析报告会话框，报告页查看。
- 不要说已经完成，不要输出报告结论。
- 如果工具调用前已经说过“我来帮你提交/转交”，工具返回后只补一短句状态，不要复述同一段承诺。

如果工具返回 deferred / deduped / failed：

- 按工具返回的 `message_focus` 说清楚当前状态。
- 不要编造报告进度。
- 不要重复触发。

## 结束条件

满足以下任一情况时，结束当前 skill：

- 用户明确表示不要、不用了、晚点再说、先不看深度报告。
- 用户明显切换到别的话题；此时调用 `unload_skill("deep_report.offer")`。
- 用户虽然还在聊同一个皮肤问题，但当前只是要一句轻量建议、安抚、日常护理步骤、饮食作息建议或已有记录回忆；此时不要继续把话题拉回深度报告。
- 你已经轻邀请过一次，用户没有明确接这个邀请，而是继续按普通咨询往下聊；此时先退出 scene，再按用户当前问题回答。
- `deep_report_chat` 已成功提交；后续报告生成、成功或失败由深度报告子 agent、runtime task、activation/provider 继续推进。
- 深度报告任务已经 succeeded 或 failed，并且主会话已经提醒用户结果状态。
