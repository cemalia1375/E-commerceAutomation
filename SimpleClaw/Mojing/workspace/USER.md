# User Profile

本文件是**用户长期稳定画像的容器**。

- base 默认为空骨架；某个 tenant 的实际画像内容由系统在 MySQL 中按对话积累、覆盖此 base。
- 你读这份文件是为了**了解用户**，而不是作为写入指令。
- 何时、如何写本文件，由 `TOOLS.md` 规定。

## 画像包含的维度（缺省可空）

- **identity**：称呼偏好、年龄段、所在地（粗粒度）
- **style**：语言偏好、回复长度偏好（简短 / 详细）、口吻偏好（直白 / 委婉）
- **skin**：稳定肤况（肤质类型、长期困扰、成分禁忌、已知过敏）
- **lifestyle**：作息与生活节奏（只在长期稳定时记）
- **goals**：长期目标-愿望（想改善 / 想达到的东西）

## 不属于本文件

- 每日肤况 / 产品使用事件 / 最近发生的事 → 由系统的 topic memory 维护
- 今日心情 / 临时状态 / 一次性细节 → 不记
- 基于你推测的画像 → 只记用户亲口表达的

## 格式

用 markdown 二级标题分段（如 `## identity`、`## skin`），段内用 `- key: value` 列出事实。同字段新值覆盖旧值，不堆列表。
